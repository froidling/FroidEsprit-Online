Raspberry Pi APRS iGate Build with RTL-SDR Blog Receiver

By Ethan | May 13, 2026

Good afternoon, folks who stumbled upon here. I just received a new SDR receiver in the mail, and have built myself an APRS iGate with my Raspberry Pi.

This post is more for myself, in case I forget any details later. I found many different resources online about it, but nothing all in one place, and there are almost certainly terminal commands I'll need to reference.

To begin with, parts. I had my old Raspberry Pi 3 Model B lying around. I used to use it for a Google AIY Voice HAT project back in the day, but the API seems to be deprecated, and I never bothered to set it up again. Second, RTL-SDR Blog V3 receiver. I went for the kit off Aliexpress, apparently the shop is official. The V4 is out and apparently better with heat management, but it doesn't come with the kit with the antennas and window mount.

After installing the Dire Wolf and RTL-SDR software, I met my first road block. Dire Wolf failed to recognize the RTL-SDR device.

So, I set it up again on my Linux laptop. Configured `direwolf.conf`, installed `rtl-sdr`, ran `direwolf`. Worked flawlessly. I was even able to log into APRSC and see the expected results. I actually still don't know why it worked here but not on my Raspberry Pi. Does PC have more robust conversion? Was it because of other software I had installed? Perhaps someone smarter than me can enlighten me.

As it happened, it turned out to be a misunderstanding involving what RTL-SDR even does. It's a data input, not an audio device. Hence why running `lsusb` showed the device, but not `arecord`, and addtionally why `rtl-test` worked as well. Browsing a few more tutorials and documentation I forgot to bookmark, I found that running `rtl_fm -f 144.39M -M fm -s 24000 - | direwolf -c sdr.conf -r 24000 -D 1 -` did the trick. Obligatory check your settings to see if this is relevant to you.

After solving this wrinkle, I ran into... another wrinkle. I could log into APRS-C, but it listed my user as 'xxx' and marked me as 'unverified'.

This was more unusual, since I made sure `direwolf.conf` was properly configured, but then I looked at the command I had entered again, it made reference to another file, `sdr.conf`. I had to set up my APRS login information there as well. That's new. I would assume it was generated after the first time I failed to log into Direwolf? I'm again unsure, someone else can let me know.

After all that, I was finally getting everything working, my station is properly showing up on the APRSC dashboard. It's not popping up on the APRS.fi map though, that's another problem I need to solve. I set up my coordinates in `direwolf.conf`, so I'm going to have to set that up another day, once I figure out how.

Software set up, the last thing to do was put it in place. I decided on the lower south window of my home; the upper window area behind the curtains had a metal bug guard that would act as an unfortunate faraday cage. A lower window isn't optimal either, but it seems to be working just fine, for now at least.

That's enough of that for now. The iGate is working as normal and is receiving packets. The only other update I have is, my domain name is now live! Note that the site is now va7ecx.ca, instead of a neocities subdomain. It has been a long time since I've used a domain name, and it's the first time I've got a .ca and not a .fun or .site or .online. It's also the first time I went with Porkbun instead of Namecheap.

I now take my leave, and shall update again next time I do something interesting.
